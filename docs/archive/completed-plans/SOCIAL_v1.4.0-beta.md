## LinkedIn

**MoJoAssistant v1.4.0-beta: Pluggable Module Architecture Complete**

I just released a significant architectural milestone on MoJoAssistant, my open-source local-first personal AI assistant project.

**What is MoJoAssistant?**
A privacy-first AI assistant that runs entirely on your own hardware. It maintains long-term memory across conversations, orchestrates background AI tasks, enforces policy boundaries on what data can leave your device, and connects to AI clients like Claude through the Model Context Protocol (MCP).

**What shipped in v1.4.0:**

The entire system is now pluggable. Every subsystem — memory, dream pipeline, persona scoring, growth tracking, skill execution, retrieval strategy, embedding backends, and storage — implements an abstract interface with a conformance test suite. Switching any backend requires only a config change. Adding a new one requires no changes to core code.

**The Agentic Bridge Pattern**

The most interesting architectural idea in this release: third-party integrations are generated by AI agents at runtime, not written by hand.

The pattern: MoJo defines the interface and conformance tests. To adopt a new embedding framework, vector database, or orchestration engine, you dispatch an agent with a "bridge installer prompt" and the target framework's documentation URL. The agent studies the external framework, writes a conformance-passing adapter, runs the test suite, and commits only if it passes. No developer glue code required.

This is the self-growing architecture I've been designing toward — a system that can expand its own capabilities by agents studying and adapting external technology against defined contracts.

**Plugin SDK**

Also shipped: `scripts/plugin_sdk.py` — a scaffold + validate CLI for third-party module authors. Any developer can generate a conformance-ready plugin skeleton for any provider type and validate it before submitting. Provider types are open-ended strings, not a closed enum — the system accepts `orchestration`, `voice`, `network`, or any new type without core changes.

**By the numbers:**
- 11 pluggable modules complete
- 293 automated tests, 0 failures
- Full CI conformance gate for every module family

**What's next:**
v1.4.1 is the setup experience — a live feature validator (`doctor --setup`), interactive MCP connectivity wizard (local / cloudflared tunnel), and stable/experimental markers on the test suite. The goal is a new user running one command and getting an honest picture of what works in their environment and exactly what to do next.

v2.0.0 drops the beta label once the setup experience is solid and the install story is clean.
