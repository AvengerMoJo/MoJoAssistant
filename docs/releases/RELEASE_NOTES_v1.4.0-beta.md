# MoJoAssistant v1.4.0-beta — Modular Architecture Complete

**Released:** 2026-05-13
**Tag:** `v1.4.0-beta`

## Theme: Pluggable Module Architecture

All 11 planned modules are complete. MoJoAssistant is now fully pluggable —
every core subsystem (memory, dream, persona, growth, skill, retrieval,
embedding, storage) is behind an ABC contract with a conformance suite.
New technology can be adopted at runtime without editing core files.

### Agentic Bridge Pattern

Third-party adapters are generated by agents at runtime, not written by hand:
- MoJo owns the ABC + conformance suite + bridge installer prompt
- An agent studies the external framework, generates the bridge, runs conformance, commits if it passes
- See `docs/architecture/AGENTIC_BRIDGE_PATTERN.md`

### Open `provider_type`

`docs/schemas/module.json` — `provider_type` changed from a closed enum to a
free `^[a-z][a-z0-9_]*$` pattern. New module types (orchestration, voice,
network) are valid without any core change. The conformance suite is the only
gate.

### CubeSandbox Integration

Reference blueprints in `config/skill_blueprints/` for `cubesandbox_exec` and
`cubesandbox_create`. Ahman (infra role) has an active task to install the
KVM/RustVMM server on the 249 host and complete end-to-end validation.
